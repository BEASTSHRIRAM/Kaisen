Kaisen Architecture

AI-Powered Security Monitoring System - System Architecture Overview

1
Data Collection Layer
Log Collector
Collects system metrics every 7 seconds (CPU, Memory, Processes)
Remote Collector
Collects data from remote systems via HTTP/HTTPS
System Commands
Executes WMIC, PowerShell, netstat for data collection
↓
2
AI Processing Layer
DQN OS Layer
Deep Q-Network for OS anomaly detection with 13 input features
DQN Agent Layer
Monitors LLM agent sessions for jailbreak detection
SHAP Explainer
Provides natural language explanations for AI decisions
Alert Engine
Generates security alerts from detected anomalies
Graph Engine
Builds attack graphs to visualize threat paths
↓
3
API & Real-Time Layer
Flask Server
REST API server providing endpoints for metrics, alerts, and graphs
Socket.IO
WebSocket server for real-time updates to connected clients
CORS Handler
Manages cross-origin requests for browser-based clients
↓
4
Frontend Presentation Layer
React Application
Main UI built with React 18, TypeScript, and Material UI
Electron Desktop
Cross-platform desktop app wrapper for native experience
D3.js Charts
Interactive attack graph visualization with force-directed layout
Chart.js
Real-time metric charts showing CPU, memory, and network trends
Zustand Store
State management for real-time data and user interactions

Data Flow

1
Collection: Log Collector gathers system metrics every 7 seconds using WMIC, PowerShell, and netstat commands
2
AI Processing: DQN models analyze metrics for anomalies, SHAP generates explanations, and Alert Engine creates security alerts
3
API Serving: Flask server provides REST endpoints and WebSocket pushes real-time updates to connected clients
4
Visualization: React frontend displays real-time metrics, interactive attack graphs, and security alerts to users
